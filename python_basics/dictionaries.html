Dictionaries¶

The last collection type we will look at is the dictionary or dict. This type consists of keys and values, which are analogous to the words and definitions in a traditional dictionary. Consider if we rewrite the chlorine list as a dictionary,

chlorine = {'chemical symbol': 'Cl', 'atomic number': 17, 'average mass number': 35.45}

type(chlorine)

dict

The meaning of each of the values is sudden a lot more clear than in the list object. We can print all of the keys, which is helping if you are passed a dictionary from another function and need to understand the contents,

chlorine.keys()

dict_keys(['chemical symbol', 'atomic number', 'average mass number'])

It is possible to access one of the values by passing the relevant key as one would pass an index,

chlorine['atomic number']

17

However, indexing does not work for dictionaries as the index is not a valid key,

chlorine[1]

---------------------------------------------------------------------------
KeyError                                  Traceback (most recent call last)
/tmp/ipykernel_1863/3992794529.py in <module>
----> 1 chlorine[1]

KeyError: 1

We can also increase the information in a dictionary by adding new keys.

chlorine['number of isotopes'] = 2

print(chlorine)

Finally, the dictionaries items need not be single variables, like floats and strings, they can also be collections,

chlorine['mass numbers'] = (35, 37)

print(chlorine)
